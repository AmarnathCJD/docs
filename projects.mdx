## Submit Your Project

Built something amazing with Gogram? We'd love to feature it!

<Steps>
  <Step title="Fork the docs repository">
    Fork the [documentation repository](https://github.com/AmarnathCJD/docs) on GitHub.
  </Step>
  
  <Step title="Add your project">
    Edit `projects.mdx` and add your project using the Card component format.
  </Step>
  
  <Step title="Create a pull request">
    Submit a PR with your project information including title, GitHub link, and description.
  </Step>
</Steps>

## Project Guidelines

<AccordionGroup>
  <Accordion title="What to Include" icon="list-check">
    - **Clear Title**: Descriptive project name
    - **GitHub Link**: Public repository URL
    - **Description**: Brief overview (1-2 sentences) of what your project does
    - **Working Code**: Ensure your project is functional and well-documented
  </Accordion>

  <Accordion title="Submission Requirements" icon="clipboard-check">
    - Project must actively use Gogram
    - Repository should be public and accessible
    - Include a README with setup instructions
    - Code should follow best practices
    - Must be actively maintained (updated within last 6 months)
  </Accordion>

  <Accordion title="Best Practices" icon="star">
    - Include example usage in your README
    - Add proper error handling
    - Document configuration options
    - Provide installation instructions
    - Include a license file
  </Accordion>
</AccordionGroup>
